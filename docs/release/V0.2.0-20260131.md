# V0.2.0 发布说明

*2026年1月31日发布*

---

本月是功能爆发式增长的一个月，重点围绕 AI 能力集成、CRM 业务模块、系统架构重构三个方向展开。完成了 AI 助手、系统设置、CRM 三大模块的开发，同时将项目从单体路由重构为模块化架构，为后续扩展打下基础。

## 新特性

### 系统设置

实现全局配置管理功能，支持 AI 配置选项卡。

### 示例模块

- 博客：新增博客详情页面、落地页，拆分路由权限（公开/受保护）。
- CRUD：包含完整数据管理功能的 CRUD 示例页面。

### AI 模块

- AI 助手模块：新增 AI 助手功能，包含智能体管理、对话界面和 API 集成。
- AI 数据助手：基于 PocketBase 数据的智能问答功能，支持数据库查询详情折叠面板。
- AI 演练场：实现 AI 聊天流式响应，支持 Markdown 渲染。

### CRM 模块

新增客户关系管理，包含客户单位管理、联系人、商机、合同管理。

### 系统框架

- 管理员模块：添加管理员仪表盘菜单和路由配置。
- Logo：添加新的 SVG logo 文件。
- 系统初始化：添加系统初始化功能及数据库迁移配置。
- 图标库：从 Hugeicons 迁移到 Heroicons。

## 功能增强

- 系统
  - 将 migration JSON 文件迁移到各模块的 migrations 目录
  - 重构管理员仪表盘，移除任务相关代码，简化用户统计展示
  - 统一使用 pb.files.getURL 方法
  - 替换确认弹窗为 AlertDialog 组件
  - 页面容器添加统一内边距
- 路由
  - 重构路由结构为模块化架构，支持模块自动注册
  - 合并 portal 模块到 examples 并重构路由结构
- 设置：重构设置页面为模块化表单组件
- 构建部署：Nginx 配置支持 LLM 接口代理，使用 envsubst 动态替换环境变量

## 缺陷修复

- 用户：修复注册页面错误处理逻辑
- 设置：修复系统设置数据集权限问题（访客无法获取全局设置）
- 构建部署：修复 LLM 接口代理配置
- 路由：修复示例页面路由路径错误
- 系统：修复未使用导入的清理
